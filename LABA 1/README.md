# Знакомство с платформой .NET. Реализация консольных приложений

Для выполнения лабораторной мы спроектировали модульную архитектуру. Выделение модулей позволяет нам делить программу на совокупность независимых блоков, структура поведения которых будет подчиняться определенным правилам. Это позволяет структурировать код, разбивая его на функционально законченные фрагменты. Модульная структура нашей программы заключается в том, что к главному модулю мы подключаем 5 модулей, в которых содержатся классы со своей функции в соответствии с заданием. В главном модуле создаются объекты этих пяти классов и происходит вызов функций.
 
#  Что понимается под термином архитектура программной системы?
Архитектура - это набор значимых решений по поводу организации системы программного обеспечения, набор структурных элементов и их интерфейсов, при помощи которых компонуется система, вместе с их поведением, определяемым во взаимодействии между этими элементами.

#  Перечислите основные разновидности архитектуры программных систем.
- Архитектура клиент/сервер 
- N-уровневая / 3-уровневая архитектура
- Многослойная архитектура 
- Проектирование на основе предметной области
- Компонентная архитектура 
- Архитектура, основанная на шине сообщений 
- Объектно-ориентированная архитектура
- Сервисно-ориентированная архитектура
#  Какие требования предъявляются к проектированию модульной архитектуры программной системы?
Модульная архитектура приложения должна обеспечивать хорошую степень расширяемости. Каждый модуль в архитектуре должен быть самодостаточным и взаимно сочетаться со всеми компонентами.

#  Перечислите основные особенности языка программирования C# и платформы .NET.
Основными особенностями языка C# являются: объектно-ориентированность, императивность, функциональность
Основные особенности платформы .NET: поддержка нескольких языков, мощная библиотека классов, разнообразие технологий.

#  Как происходит компиляция и выполнение программы на языке C#? 
Компилятор Roslyn конвертирует код на языке С# в код на промежуточном языке IL и сохраняет его в сборке. Во время работы CoreIL загружает IL-код из сборки, динамически компилирует в машинные инструкции для процессора, после чего процессор выполнет код.
